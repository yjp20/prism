====test====
When I send a request to an operation with Accept header containing */*
And this operation does not produce a response with content-type of */*
Then I should get back a response with the first content-type from the list
====spec====
{
  "info": {
    "_postman_id": "37c4cd69-28e5-48aa-8386-a4ec010216e4"
  },
  "item": [
    {
      "id": "3b4f5a70-2a8f-46ba-a04b-a6f80621ad3f",
      "request": {
        "url": {
          "path": ["todos"],
          "host": ["/"]
        },
        "method": "GET"
      },
      "response": [
        {
          "description": "Get Todo Items",
          "code": 200,
          "header": [
            {
              "key": "Content-Type",
              "value": "text/plain"
            }
          ],
          "body": "hello"
        },
        {
          "description": "Get Todo Items",
          "code": 200,
          "header": [
            {
              "key": "Content-Type",
              "value": "application/json"
            }
          ],
          "body": "\"hello\""
        }
      ]
    }
  ]
}
====server====
mock -p 4010 ${document}
====command====
curl -i -X GET http://localhost:4010/todos -H "accept: */*"
====expect====
HTTP/1.1 200 OK
content-type: text/plain
content-length: 5
Connection: keep-alive

hello
